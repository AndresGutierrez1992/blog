NodeJS-Desarrolo Web

Node.js es un entorno de ejecución para JavaScript que permite ejecutar código en el servidor. Fue creado sobre el motor V8 de Google Chrome.
Con Node.js puedes crear servidores web, APIs REST, trabajar con bases de datos y mucho más. Su ecosistema, con npm como gestor de paquetes.
Uno de sus principales beneficios es que permite a los desarrolladores usar JavaScript en el backend, lo cual unifica el lenguaje tanto en cliente como en servidor.

Diferencias entre JavaScript en el Frontend y en el Backend

Frontend: Se ejecuta en el navegador. Se usa para manipular el DOM, manejar eventos de usuario, y actualizar la interfaz visual de una página web.

Backend: Se ejecuta en el servidor (con Node.js). Se encarga de la lógica de negocio, conexión con bases de datos, autenticación, y procesamiento de datos.

Ventajas y desventajas de usar JavaScript en el Backend

Ventajas: Uso de un solo lenguaje (JS) para frontend y backend, gran comunidad, no bloqueante (event loop), ideal para APIs.

Desventajas: No tan eficiente como otros lenguajes en operaciones intensas de CPU, menos robusto en comparación con lenguajes como Java o C#.

    • ¿Qué es npm y quién lo mantiene?

    NPM (Node Package Manager) es el sistema de gestión de paquetes para Node.js. Permite instalar bibliotecas y herramientas. Es mantenido por GitHub y la comunidad de Node.js.

    ¿Qué es Yarn y en qué se diferencia de npm?

    Yarn es otro gestor de paquetes creado por Facebook. Es más rápido en instalaciones debido a su caché y trabaja con un enfoque más estricto con dependencias.

    Diferencias entre yarn.lock y package-lock.json

    • Formato: yarn.lock es para Yarn, package-lock.json es para npm.
    • Compatibilidad: yarn.lock solo es útil con Yarn; package-lock.json solo con npm.

    5 comandos útiles de npm

    • npm init: Crea un archivo package.json.
    • npm install: Instala dependencias listadas.
    • npm install express: Instala una librería.
    • npm uninstall express: Elimina una librería.
    • npm update: Actualiza todas las librerías.

    ¿Para qué sirven estas flags?

    • -g: Instala globalmente el paquete.
    • --save: Guarda la dependencia en package.json (ya no se necesita desde npm 5).
    • -D: Instala como dependencia de desarrollo.

    ¿Para qué sirven estos módulos?

    • express: Express es un framework para Node.js que te ayuda a crear servidores web de forma rápida,
    • mongodb: Conexión con MongoDB.
    • mongoose: ODM para MongoDB. (Object Document Mapper) es una herramienta que se utiliza para trabajar con bases de datos NoSQL,
      como MongoDB, de una forma más sencilla y organizada.
    • path: Manipulación de rutas de archivos.
    • router: Manejo de rutas (parte de Express).
    • nodemon: Reinicio automático del servidor.
    • cors: Permitir solicitudes entre dominios.
    • http: Crear servidor HTTP básico.
    • multer: Carga de archivos.
    • morgan: Logs de peticiones HTTP.
    • prettier: Formateador de código.
    • debug: Sistema de logs. librería de Node.js que te ayuda a ver mensajes en la consola para entender qué está pasando dentro de tu app.
    • jade: Motor de plantillas (ahora llamado Pug).
    • passport: Para autenticar usuarios en tu app: Login con usuario y contraseña, Login con Google, Facebook, GitHub, Twitter, etc; Mantener sesiones de usuario (saber quién está logueado).

    Validación, PDF, Excel

    • Validar formularios: validator, express-validator
    • Exportar a PDF: pdfkit, puppeteer
    • Exportar a Excel: exceljs
    • Leer Excel/PDF: xlsx, pdf-parse

    3 módulos interesantes

    • chalk: Colorea texto en la terminal.
    • figlet: Se puede usar en proyectos de Node.js para imprimir mensajes bonitos en la terminal o incluso en páginas web para mostrar texto con estilo retro de ASCII art.
    • dotenv: Maneja variables de entorno. Son valores que usas para configurar tu app, como contraseñas, claves API, URLs de bases de datos, puertos, etc.

    ¿Cómo usar un módulo instalado?

    Luego de instalar con npm install nombre, se importa con require('nombre').

    ¿Cómo instalar solo para desarrollo?

    Usa npm install nombre --save-dev o npm install nombre -D.
